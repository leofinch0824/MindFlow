## 1. MVP 目标

本轮 MVP 的主目标是先让用户稳定完成以下核心链路：

**配置 AI → 查看本周日报 → 阅读 Main Channel → 通过反馈纠偏兴趣 → 系统记录学习信号**

本清单只覆盖当前已确认的 MVP 范围，不扩展到后续推荐解释、强抑制模式或更复杂的多区块反馈体系。

---

## 2. 已确认的产品决策

### 2.1 日报浏览规则

- 默认进入 Newsletter 页面时，只查询**当前所在周**的日报列表。
- 日期选择器升级为**周切换面板**，不再是简单日期下拉列表。
- 支持在日期选择器中**向前翻周**，历史数据按周 **lazy load**。
- 周面板中：
  - **有简报日期**可点击；
  - **无简报日期**灰色不可点击。
- 若用户当天打开页面但**今日暂无简报**：
  - 页面默认显示“今日暂无简报”状态；
  - 同时提供一个可点击的提示文本，跳转到**最近一个有简报的日期**。

### 2.2 Main Channel 反馈规则

- 本轮 MVP **只在 Main Channel 提供反馈能力**。
- 不使用“点赞 / 点踩”双按钮。
- 只保留一个动作：**“减少这类话题内容”**。
- 该动作语义为：
  - 这条内容与我的兴趣**不够相关**；
  - 这是一个**未来推荐排序的学习信号**；
  - 不是对文章来源质量的评价，也不是对来源本身的否定。
- 用户点击该动作后：
  - 当前卡片**淡出并移出列表**；
  - 后续卡片**自动补位**；
  - 页面底部弹出 **snackbar / toast**；
  - snackbar 提供 **Undo / 撤销**。
- 本轮 MVP **不额外请求新内容补位**，只做当前列表内重排。
- 本轮 MVP **不向用户解释**具体哪些兴趣标签、权重或排序参数发生了变化。
- 本轮 MVP **不引入强抑制模式**，即连续多次负反馈也不触发“彻底屏蔽某类话题”的特殊逻辑。

### 2.3 Read Source 行为规则

- `Read Source` 点击行为记为**弱隐式兴趣信号**。
- 上报方式为**非阻塞埋点**。
- 埋点失败时：
  - 可以记录日志；
  - **绝不能阻断原文跳转**。

### 2.4 Settings 页面规则

- `Test Connection` 默认测试**当前草稿配置**，而不是后端已保存配置。
- `Save Architecture` 的执行顺序固定为：
  1. 先自动验证当前草稿；
  2. 验证通过后再保存；
  3. 验证失败则阻止保存并提示错误。
- `api_key` 不回显真实值。
- `api_key` 保存规则：
  - **首次配置**时必须填写；
  - **已配置情况下留空**表示保留原有 key；
  - **输入新 key**表示尝试替换，必须验证通过后才能写入数据库。
- 本轮 MVP **不提供“清空 API Key”按钮**。
- Settings 页面必须支持 4 态：
  - `loading`
  - `load_error`
  - `unconfigured`
  - `configured`
- `load_error` 必须显式显示“加载失败，可重试或重新配置”。
- `unconfigured` 状态采用：
  - 顶部说明提示块；
  - 直接展示可编辑表单；
  - 不增加“开始配置”中间步骤。

## 5. 本轮 MVP 非目标

以下事项明确**不纳入本轮 MVP**：

- Exploration Zone / Surprise Box 反馈按钮
- 点赞 / 点踩双按钮
- 向用户解释标签权重变化
- 清空 API Key 按钮
- 实时补新卡片
- 强抑制 / 硬屏蔽话题逻辑
- “内容质量差”反馈维度
- 完整月历式日期选择器

---

## 6. MVP 完成定义

满足以下条件，才可声明本轮 MVP 完成：

1. 用户可稳定配置 AI，且不会误覆盖 `api_key`。  
2. Settings 页的 `Test Connection` / `Save Architecture` 语义一致。  
3. 日报页支持按周浏览，且无简报日期不会误点。  
4. 今日无简报时，状态透明且可跳转到最近一份日报。  
5. Main Channel 的“减少这类话题内容”完整可用。  
6. `Read Source` 能记录弱隐式兴趣信号。  
7. 后端关键稳定性问题已修复。  
8. 删除兴趣标签不会因 `204 No Content` 导致前端报错。  

### 当前对照结论

截至 2026-04-12，上述 8 条完成定义均已覆盖到对应实现或联调验证，当前可以把本轮 MVP 判断为 **主链路已完成**。后续工作重点转入：

- 继续做真实数据下的 UI/容器联调；
